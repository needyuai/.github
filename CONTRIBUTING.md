# Contribuindo

Obrigado pelo interesse em contribuir com os projetos do ecossistema **Tech Human · Trustyu.ai · NeedyU.ai**.

## Repositórios públicos

A maioria dos nossos repositórios é **privada** — propriedade intelectual de clientes e ativos próprios. As contribuições externas, por enquanto, são bem-vindas nos repositórios públicos listados na [home da organização](https://github.com/needyuai).

## Antes de contribuir

1. Leia o nosso [Código de Conduta](./CODE_OF_CONDUCT.md)
2. Leia a [política de segurança](./SECURITY.md) — vulnerabilidades **nunca** devem entrar como issue pública
3. Verifique se já existe uma issue ou PR aberta para o que você quer fazer

## Fluxo de contribuição

1. **Abra uma issue antes do PR** — descreva o problema, proposta ou melhoria. Isso evita retrabalho e alinha expectativas
2. **Fork e branch** — trabalhe em uma branch descritiva (`feat/...`, `fix/...`, `docs/...`)
3. **Commits** — mensagens claras, no imperativo, em português ou inglês
4. **Testes** — se o repositório tem suíte de testes, garanta que tudo passe
5. **Abra o PR** — referencie a issue (`Closes #123`), descreva o que foi feito e por quê

## Padrões

- **Linguagem** — pt-BR e en-US são bem-vindos; siga o padrão do repositório
- **Estilo de código** — siga o linter/formatter configurado no projeto
- **Documentação** — toda mudança que afeta uso público deve atualizar README ou docs

## Revisão

PRs passam por revisão de um mantenedor. Pode haver pedidos de ajuste — encare como conversa, não como rejeição.

## Licenciamento

Ao enviar uma contribuição, você concorda que ela será licenciada sob os mesmos termos do repositório receptor.

## Dúvidas

- Comerciais ou de produto → [`contato@trustyu.ai`](mailto:contato@trustyu.ai)
- Técnicas sobre um repo específico → abra uma issue no repositório em questão
- Segurança → [`security@trustyu.ai`](mailto:security@trustyu.ai)
